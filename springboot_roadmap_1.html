Why Kafka:  When service A needs to notify service B without waiting for B to respond, you use a message queue. Kafka decouples services — student-service publishes a "student.registered" event; notification-service consumes it asynchronously. Neither waits for the other.

Memory Trick — How to NOT Forget

  • Kafka vs REST: REST = phone call (synchronous, caller waits). Kafka = text message (asynchronous, sender moves on immediately). Use Kafka when the producer must not wait for the consumer.
  • Offset = bookmark in a book. Consumer remembers which message it last read. If it crashes, it can resume from the same offset — no message is lost.
  • Consumer Group rule: if you have 3 partitions and 3 consumers in the same group, each consumer gets exactly 1 partition. Adding a 4th consumer means one is idle. Partitions ≥ consumers in group for maximum parallelism.
  • DLT is your safety net: never let a bad message block your entire queue. Always configure retries + DLT in production.
