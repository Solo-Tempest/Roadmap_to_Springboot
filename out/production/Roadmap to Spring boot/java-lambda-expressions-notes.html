What is a Lambda Expression?

A Lambda Expression is an anonymous function — a function that has no name, no explicit return type declaration, and no access modifier. It was introduced in Java 8 (Java 1.8).

⭐ Java Version — Important! Lambda expressions were introduced in Java 1.8. They cannot be used in Java 1.7, 1.6, or earlier versions. Along with Streams, they are the most important feature added in Java 8.

Anonymous Function — Definition

A normal function (method) has four parts: modifier, return type, name, and body. An anonymous function has none of the first three — only the body matters.

// Transforming a normal method → Lambda
STEP 1 — Normal method
public void greet() { System.out.println("Hello"); }
Remove modifier, return type, name → add arrow symbol ->
STEP 2 — Lambda
() -> { System.out.println("Hello"); }
💡 Simple Definition Lambda = parameters + arrow symbol + body    i.e.    (params) -> { body }
2

Syntax — How to Write a Lambda

The general syntax of a lambda expression is:

Lambda Syntax
// General form
(parameters) -> { body }

// ── Example 1: No parameters, no return ──
() -> { System.out.println("Hello from Lambda!"); }

// ── Example 2: With parameters, returns value ──
(int a, int b) -> { return a + b; }

// ── Example 3: Single parameter ──
(String s) -> { return s.length(); }

Full Example — Normal Method to Lambda

Normal Method vs Lambda
// ── Normal method that adds two numbers ──
public int sum(int a, int b) {
    return a + b;
}

// ── Step 1: Remove modifier (public) and return type (int) ──
// ── Step 2: Remove method name (sum) ──
// ── Step 3: Add arrow symbol (->) after parameters ──

// ── Lambda version ──
(int a, int b) -> { return a + b; }

// ── Shortest version (single expression — see Section 3) ──
(a, b) -> a + b
3

Syntax Rules & Shortcuts

There are two important shortcut rules that make lambdas even shorter:

Rule 1 — Single Statement: Remove braces & return

If the lambda body contains only one statement, you can remove the curly braces {} and the return keyword.

Rule 1 — Single Statement Shortcut
// Full lambda — with braces and return
(int a, int b) -> { return a + b; }

// Shortcut — single statement, remove {} and return
(int a, int b) -> a + b

// Multiple statements — MUST use braces
(int a, int b) -> {
    System.out.println("Adding...");
    return a + b;   // return keyword required here
}

Rule 2 — Type Inference: Remove parameter types

The Java compiler can automatically infer the type of parameters from the functional interface. So you don't need to write types explicitly.

Rule 2 — Type Inference
// With types written explicitly
(int a, int b) -> a + b

// Without types — compiler infers from the functional interface
(a, b) -> a + b      // ✅ same result, shorter!

// String parameter — type inferred automatically
// Full:    (String s) -> { return s.length(); }
// Short:   s -> s.length()
s -> s.length()

// How does the compiler infer types?
// → It reads the functional interface the lambda is assigned to
// → The method signature of that interface tells it the types
💡 How Type Inference Works When you assign a lambda to a functional interface variable, the compiler looks at the abstract method of that interface, reads its parameter types, and applies them automatically to your lambda. You never need to re-type them.

Complete Shortcut Progression

From Verbose to Minimal
// Level 1 — Full lambda
(int a, int b) -> { return a + b; }

// Level 2 — Remove braces + return (single statement)
(int a, int b) -> a + b

// Level 3 — Remove types (type inference)
(a, b) -> a + b    // ← most concise!

Benefits of Lambda Expressions

1 — Reduces Lines of Code

A task that previously required creating a class, implementing an interface, and overriding a method can now be done in a single line.

2 — Can Only Be Used with Functional Interfaces

Lambdas are purpose-built for functional interfaces (interfaces with exactly one abstract method). This keeps their usage clean and predictable.

3 — Enables Passing Behavior as a Parameter

You can pass a lambda (a function) as an argument to another method. This is called behavior parameterization and is the core of functional programming.

Passing Lambda as Argument
// You can pass a lambda directly to a method expecting a functional interface
someMethod((a, b) -> a + b);   // behavior passed inline, no class needed

4 — Enables Functional Programming in Java

Before Java 8, Java was purely Object-Oriented. Lambdas introduced the ability to treat functions as first-class citizens — passing them around just like objects. This is a major paradigm shift.

5 — More Readable and Maintainable Code

Code becomes shorter, cleaner, and easier to understand. Less boilerplate means fewer bugs and faster development.

✅ Summary of Benefits Reduces boilerplate code · Supports passing behavior · Enables functional programming · Makes code more readable · Works seamlessly with Java's Collections and Streams API.
📌 Historical Context Other programming languages (like Python, JavaScript, Scala) had lambda/anonymous function support long before Java. Java 8 finally brought this capability to Java, making it competitive with modern functional languages.

Functional Interface — The Key Requirement

A lambda expression can only be used with a Functional Interface. This is the most important rule to remember.

⭐ Definition — Functional Interface A Functional Interface is any interface that has exactly one abstract method. It can have default or static methods, but only ONE abstract method.

Built-in Functional Interfaces in Java

Java already provides many functional interfaces. The most common ones you will encounter are:

InterfacePackageAbstract MethodUsed For
Runnablejava.langrun()Thread tasks
Comparablejava.langcompareTo()Sorting/comparing
ActionListenerjava.awtactionPerformed()GUI button events
Callablejava.util.concurrentcall()Tasks with return value

@FunctionalInterface Annotation

You can mark your interface with @FunctionalInterface to tell the compiler to enforce the single-method rule. If you accidentally add a second method, the compiler will give an error.

@FunctionalInterface
@FunctionalInterface           // optional but recommended
interface MyInter {
    void sayHello();            // only ONE abstract method allowed
    // void anotherMethod();   ❌ compiler error — breaks @FunctionalInterface
}
⚠️ Without @FunctionalInterface The annotation is optional — the interface still works as a functional interface with just one method. But adding @FunctionalInterface gives you compile-time safety. Without it, someone could accidentally add another method and break all lambdas using it.

Creating Your Own Functional Interface

You can create a custom functional interface and use a lambda to implement it. Here is a complete example:

Example 1 — Simple Greeting

Custom Functional Interface — No Parameters
// Step 1: Create the functional interface
@FunctionalInterface
interface MyInter {
    void sayHello();
}

// Step 2: Use a lambda to implement it
public class Main {
    public static void main(String[] args) {

        // Lambda assigned to functional interface reference
        MyInter i1 = () -> System.out.println("Hello from Lambda!");

        // Call the method via the reference
        i1.sayHello();

        // Reuse the reference with a different lambda
        MyInter i2 = () -> System.out.println("Second time using lambda!");
        i2.sayHello();
    }
}
OUTPUT
Hello from Lambda! Second time using lambda!

Example 2 — With Parameters & Return Value

Custom Functional Interface — Parameters + Return
// Functional interface that takes two ints and returns an int
@FunctionalInterface
interface MathInter {
    int operate(int a, int b);
}

public class Main {
    public static void main(String[] args) {

        // Lambda for addition
        MathInter add = (a, b) -> a + b;

        // Lambda for multiplication
        MathInter mul = (a, b) -> a * b;

        System.out.println(add.operate(5, 3));   // 8
        System.out.println(mul.operate(5, 3));   // 15
    }
}
OUTPUT
8 15

Example 3 — String Length

String Length via Lambda
@FunctionalInterface
interface LengthInter {
    int getLength(String s);
}

LengthInter li = s -> s.length();   // type of s inferred from interface

System.out.println(li.getLength("Hello"));   // 5
System.out.println(li.getLength("Lambda"));  // 6
OUTPUT
5 6

Three Ways to Implement a Functional Interface

Before lambdas, there were two standard ways to implement an interface. Lambda is the third and cleanest way.

Way 1 — Separate Class (old way)

Way 1 — Separate Class
// Create a separate class that implements the interface
class MyInterImpl implements MyInter {
    @Override
    public void sayHello() {
        System.out.println("Hello from class!");
    }
}

// Use it
MyInter obj = new MyInterImpl();
obj.sayHello();

// Disadvantage: Need to create a whole new file/class just for one method

Way 2 — Anonymous Class (less verbose)

Way 2 — Anonymous Class
// No need to create a separate file — implement inline
MyInter obj = new MyInter() {
    @Override
    public void sayHello() {
        System.out.println("Hello from anonymous class!");
    }
};                // ← semicolon here!

obj.sayHello();

// Better — no separate class file, but still verbose

Way 3 — Lambda (best way ✅)

Way 3 — Lambda Expression
// Clean, concise, one line!
MyInter obj = () -> System.out.println("Hello from Lambda!");
obj.sayHello();
WayCode NeededSeparate File?Recommended?
Separate ClassMost verboseYes❌ Old style
Anonymous ClassMediumNo⚠️ Okay
LambdaMinimalNo✅ Best

Lambda with Threads (Runnable)

Runnable is a functional interface with one method: run(). This makes it a perfect candidate for lambdas when creating threads.

Old Way — Anonymous Class with Thread

Thread — Old Way (Anonymous Class)
Thread t1 = new Thread(new Runnable() {
    @Override
    public void run() {
        for (int i = 1; i <= 5; i++) {
            System.out.println("Thread 1: " + i);
        }
    }
});
t1.start();

New Way — Lambda with Thread

Thread — Lambda Way
// Thread 1 — prints 1 to 10
Thread t1 = new Thread(() -> {
    for (int i = 1; i <= 10; i++)
        System.out.println("Friend: " + i);
});

// Thread 2 — prints multiplication table of 2
Thread t2 = new Thread(() -> {
    for (int i = 1; i <= 10; i++)
        System.out.println("2 × " + i + " = " + (2 * i));
});

t1.setName("Thread-1");
t2.setName("Thread-2");
t1.start();
t2.start();   // both run concurrently
OUTPUT (order may vary — concurrent threads)
Friend: 1 2 × 1 = 2 Friend: 2 2 × 2 = 4 ...etc (interleaved)
✅ Why Lambda Works Here Runnable is a functional interface — it has only one method run(). The lambda body becomes the implementation of run(). No extra class needed, no boilerplate.

Lambda in GUI — ActionListener Example

ActionListener is a functional interface from Java's Swing GUI library. It has one method: actionPerformed(ActionEvent e). It is triggered when a user clicks a button.

Old Way — Anonymous Class

ActionListener — Old Way
button.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        JOptionPane.showMessageDialog(null, "Button Clicked!");
    }
});   // verbose and boilerplate-heavy

New Way — Lambda

ActionListener — Lambda Way
// Lambda replaces the entire anonymous class
button.addActionListener(e ->
    JOptionPane.showMessageDialog(null, "Button Clicked!")
);

// ── Full Swing Example ──
import javax.swing.*;
import java.awt.*;

public class MyWindow {
    public static void main(String[] args) {
        JFrame frame = new JFrame("My Window");
        JButton button = new JButton("Click Me");

        // Lambda handles button click event
        button.addActionListener(e ->
            JOptionPane.showMessageDialog(null, "Button Clicked!")
        );

        frame.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        frame.setSize(400, 300);
        frame.setLayout(null);
        button.setBounds(150, 100, 100, 40);
        frame.add(button);
        frame.setVisible(true);
    }
}
⚠️ Lambda Limitation — Multi-Method Interfaces If an interface has more than one abstract method (e.g., MouseMotionListener has mouseDragged() and mouseMoved()), you CANNOT use a lambda. You must use an anonymous class or an adapter class instead. Lambda only works with functional interfaces (exactly 1 abstract method).

⚠️ Common Pitfalls — Never Forget

1 — Lambda requires a Functional Interface You cannot store a lambda in just any variable. It must be assigned to a functional interface type. Without a target functional interface, the compiler won't know what type the lambda is.
2 — Multi-method interfaces block lambdas If an interface has 2 or more abstract methods (e.g., MouseMotionListener), you cannot use a lambda. Use an anonymous class or adapter class instead.
3 — Don't mix up shortcut rules You can only remove {} and return when the body is a single expression. If there are multiple lines, braces are mandatory and return must be written explicitly.
4 — Java version matters Lambdas only work in Java 8 (1.8) and above. If your project uses Java 7 or earlier, lambdas will cause a compilation error.
5 — @FunctionalInterface is optional but recommended Not using it means someone can accidentally add a second method to your interface, silently breaking all lambdas that use it. Always annotate custom functional interfaces.
